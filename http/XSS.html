# XSS 攻击

针对日趋复杂的 web 应用，web 安全问题逐渐暴露出来，也越来越引起开发者的重视。在大多数黑客的攻击案例中，XSS 和 CSRF 攻击是被利用的最多的方式。本文意在简单地了解 XSS 攻击方式。参考链接

XSS 名为“跨站脚本攻击”（cross-site-script），因为首字母缩写和 CSS 样式扩展名重叠，故称为 XSS。

XSS 攻击分为三种类型：反射型（非持久型）、存储型（持久型）、基于 DOM。

# XSS 攻击方式

攻击者依靠一切可能的手段，想方设法在受害者的浏览器中执行自己的“有毒”脚本，例如窃取一些用户的敏感数据，把 cookie、session 发送给攻击者，将受害者重定向到由攻击者控制的网站，然后在受害者的机器上做一些恶意操作。

# 持久型 XSS

演示

  • 攻击者将恶意代码提交到目标网站的数据库中
  • 用户打开目标网站时，网站服务端将恶意代码从数据库取出，拼接在 HTML 中返回给浏览器
  • 用户浏览器接收到响应后，混在其中的恶意代码也被执行了
  • 恶意代码窃取用户数据并发送给攻击者，或者冒充用户的行为，调用目标网站的接口执行恶意操作

这种攻击常见于带有保存用户数据的网站，例如：论坛发帖、商品评论、用户私信等。

# 非持久型 XSS

演示

  • 攻击者构造特殊的 URL ，其中包含恶意代码
  • 用户打开带有恶意代码的 URL 时，网站服务端将恶意代码取出，拼接在 HTML 中返回给浏览器
  • 浏览器收到响应后解析执行，混在其中的恶意代码也被执行
  • 恶意代码窃取用户数据并发送给攻击者，或者冒充用户的行为，调用目标网站的接口执行恶意操作

非持久型 XSS 漏洞常见于通过 URL 传递参数的功能，如网站搜索、跳转等。

# DOM 型 XSS

  • 攻击者构造出特殊的 URL ，其中包含恶意代码
  • 引诱用户打开带有恶意代码的 URL
  • 前端 JS 取出 URL 中的恶意代买并执行
  • 恶意代码窃取用户数据并发送给攻击者，或者冒充用户的行为，调用目标网站的接口执行恶意操作

DOM 型 XSS 跟前两种 XSS 的区别： DOM 型的漏洞中，取出和执行恶意代码是由浏览器完成的，属于前端自身的安全漏洞，前面两种属于服务端的安全漏洞。

# 如何防范

  • HttpOnly 防止劫取 cookie

    浏览器将禁止页面的 Javascript 访问带有 HttpOnly 属性的 Cookie。

    严格来说，HttpOnly 并非阻止 XSS 攻击，而是能阻止 XSS 攻击后的 Cookie 劫持攻击。

    //设置cookie
response.addHeader("Set-Cookie", "uid=112; Path=/; HttpOnly")

//设置https的cookie
response.addHeader("Set-Cookie", "uid=112; Path=/; Secure; HttpOnly");

//设置后读取 cookie 的方法
Cookie cookies[]=request.getCookies();

  • 对输入/输出进行转义

    不相信用户的任何输入，对于用户的输入要进行 检查、转义 、过滤 ，例如对 & < > " ' / 这几个字符进行转义
