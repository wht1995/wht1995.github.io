# HTTP 常见状态码

当浏览者访问一个网页时，浏览者的浏览器会向网页所在服务器发出请求。当浏览器接收并显示网页前，此网页所在的服务器会返回一个包含HTTP状态码的信息头（server header）用以响应浏览器的请求。

下面是常见的HTTP状态码：

  • 1xx：表示服务器已收到请求，需要请求者继续处理。
  • 2xx：表示请求已成功被服务器接收、理解、并接受。
  • 3xx：重定向，请求资源的位置已发生变动，需要客户端进一步（重新）请求。
  • 4xx：客户端请求错误，请求包含有错误的语法或无法完成请求。
  • 5xx：服务器错误，服务器在处理请求的过程中发生了异常。

# 1XX

100 Continue ：通知客户端它的部分请求已被接收，且仍未被拒绝。客户端应当继续发生请求

101 Switching Protocols ：切换协议。服务器根据客户端的请求切换协议，只能切换到更高级的协议，例如切换到HTTPS

# 2XX

200 Ok ：请求已成功，会在响应报文中出现，多用于 GET 和 POST

201 Created ：请求已被实现，且有新的资源依据请求的需要而被建立

202 Accept ：服务器已接受请求，但仍未处理

204 No Content ：服务器已成功处理请求，但不需要返回任何内容

206 Partial Content ：服务器已成功处理了部分请求。类似迅雷这类的 HTTP下载工具都是使用此类响应实现断点续传或者将一个大文档分解为多个下载段同时下载。响应会包含 Content-Range 、Date等头部域

# 3XX

301 Moved Permanently ：永久重定向。

302 Move Temporarily ：临时重定向。

301 与 302 的共同点 ：都表示重定向，就是说，浏览器在拿到服务器返回的这个状态码后会自动跳转到新的 URI 地址，这个地址可以从响应的 Location 头部获取（用户看到的效果就是他输入了 A 地址瞬间就变成了重定向后的 B 地址）。

301 与 302 的不同点 ：网址 A 做了一个 302 重定向到网址 B，这时候，搜索引擎显示的网址仍然是 A 的网址，但网页上显示的内容却是 B 的内容，这种情况称之为”URL 劫持“。

  • 301 表示旧的地址已被移除（该资源不可访问了），搜索引擎在抓取新内容时把地址也换成了重定向后的地址
  • 302 表示旧地址 A 的资源还在，仍可访问。搜索引擎会抓取新内容而保留旧地址，只是临时跳转地址而已
  • 301 会做缓存优化，302 不会

304 Not Modified ：未修改。所请求的资源未修改，不会返回任何资源。客户端可通过缓存访问资源（协商缓存）。

# 4XX

400 Bad Requested ：客户端的请求语法或参数错误，服务端无法理解

401 Unauthorized ：当前请求需要用户验证

403 Forbidden ：服务器理解客户端请求，但是拒绝执行请求。比如 IIS 或者 apache 设置了访问权限不当

404 Not Found ：请求的资源在服务器上不存在（找不到）

405 Not Allowed ：请求行中指定的请求方法不能被用于请求相应的资源

406 Not Acceptable : 请求的资源的内容特性无法满足请求头中的条件，因而无法生成响应实体

408 Request Timeout : 服务器等待客户端发送的请求时间过长，超时

413 Request Entity Too Large : 由于请求的实体过大，服务器无法处理，因此拒绝请求

# 5XX

500 Internal Server Error ：服务器内部错误，无法完成请求

501 Not Implemented ：服务器不支持当前请求所需要的某个功能

502 Bad Gateway ： 作为网关或者代理工作的服务器尝试执行请求时，从远程服务器接收到了一个无效的响应

503 Service Unavailable ：服务器当前繁忙，无法响应请求
