    # BFC 原理及其简单应用

    本文权当记录学习，结合自己所看的文章资料以及理解成文。

    本文主要记录以下几点：

    • 什么是 BFC
    • 如何创造（产生）一个 BFC 的渲染效果
    • 如何简单使用 BFC

    # 什么是 BFC

    ​ BFC 的全称是 Block Formatting Context ，翻译过来即 块级格式上下文 。是 CSS 2.x 的一个规范。我们知道在一个页面的布局中，我们会使用到 块级元素 和 行内元素 。块级元素 一般是容器级别的，例如 <div> 、<p>、<h>系列、<li>等。 本文 BFC 多指 <div> 元素，它是一个独立的渲染区域，有自己的一套渲染规则，它规定了其内部子元素如何布局，并且该区域内部布局不受外部影响，也不影响外部。

    我自己的理解就是，把 BFC 看作一个大箱子，里面的布局如何如何，即使天翻地覆，都不会影响它外部的布局，也不受外部影响。你强任你强，老子...😄

    # 如何创建 BFC

    • 根元素 或其它包含它的元素

    • 浮动 float: left | right （属性值不为 none 时）

    • 元素定位 position: absolute | fixed（脱离文档流时）

    • 元素 display: inline-block | table-cell | table |

    • overflow 的值不为 visible 的时候（hidden | auto | scoll）

    • 弹性盒子 flexBox(元素的display: flex或inline-flex)

      常见的就是overflow:hidden、float:left/right、position:absolute。也就是说，每次看到这些属性的时候，就代表了该元素以及创建了一个 BFC 了。

    # BFC 的特性

    • 内部的盒会在垂直方向一个接一个排列（与普通文档流布局相似）
    • 属于同一个 BFC 的两个相邻Box的 margin 会发生重叠（塌陷），与方向无关
    • 每个元素的左外边距与包含块的左边界相接触（从左向右），即使浮动元素也是如此。（这说明BFC中子元素不会超出他的包含块，而 position 为 absolute 的元素可以超出他的包含块边界）
    • BFC 的区域不会与 float 的元素区域重叠
    • 计算 BFC 的高度时，考虑 BFC 所包含的所有元素，连浮动元素也参与计算
    • BFC 就是页面上的一个隔离的独立容器，容器里面的子元素不会影响到外面元素，反之亦然

    # BFC 的应用

    • 防止 margin 重叠/塌陷
    <style>
	.block {
		background: pink;
		width: 200px;
		line-height: 100px;
		text-align:center;
		margin: 100px;
	}
</style>
<body>
	<div class="block">Block 1</div>
	<div class="block">Block 2</div>
</body>

    ​ 根据 BFC 特性第二条：

    属于同一个 BFC 的两个相邻 Box 的 margin 会发生重叠（塌陷），与方向无关

    ​ 对于这段代码，<body> 及其包含的两个 <div> 形成了一个 BFC ，它满足了创建生成 BFC 的一个条件。接着，在这个 BFC 里面，两个 <div> 的 margin 发生了折叠（坍塌），我们原本希望两个 <div> 之间的 margin 是 200px ，但是目前只有 100px（以最大的那个值为准），也就是说，如果 Block2 的 margin 是 80px 的话，两个 <div> 的间距还是以 100px 为准。那么我们为了让两个 <div> 的间距变为 200px ，我们还要在内部再创建一个 BFC 。

    ​ 我们要在其中一个 <div> 的外面再包裹一层块级容器，这样就会生成一个新的 BFC ，如此一来，便不会发生 margin 重叠（坍塌）。

    <style>
	.block {
		background: pink;
		width: 200px;
		line-height: 100px;
		text-align:center;
		margin: 100px;
	}
	.wrap{
		overflow: hidden //增加此属性
	}
</style>
<body>
	<div class="block">Block 1</div>
	<div class="wrap">
		<div class="block">Block 2</div>	
	</div>
</body>

    ​ 我们在 Block2 的外面包裹了一层新 <div class = "wrap"> ，并且设置其属性为 overflow: hidden ，如此一来便会形成一个新的 BFC ，两个 Block 之间就不会发生 margin 重叠（坍塌）。

    这里设置 float: left | right ，position: absolute | fixed，等都会生成新的 BFC 防止 margin 坍塌。

    • 清除内部浮动

      <style>
    .box {
        border: 5px solid #fcc;
        width: 300px;
    }
 
    .child {
        border: 5px solid green;
        width:100px;
        height: 100px;
        float: left;
    }
</style>
<body>
    <div class="box">
        <div class="child">child 1</div>
        <div class="child">child 2</div>
    </div>
</body>

      ​ 我们看到，由于子元素设置了浮动，所以父容器并没有能包裹住两个 <div> 。但是，我们想要的效果是父容器的框能包裹住两个子元素。怎么实现呢？

      <style>
    .box {
        border: 5px solid #fcc;
        width: 300px;
        overflow: hidden // 增加这条属性
    }
 
    .child {
        border: 5px solid green;
        width:100px;
        height: 100px;
        float: left;
    }
</style>
<body>
    <div class="box">
        <div class="child">child 1</div>
        <div class="child">child 2</div>
    </div>
</body>

      ​ 为了清楚父容器内部的浮动，我们可以使父容器产生一个 BFC 的效果，如此一来，父容器 box 在计算高度的时候就会包裹子元素了，内部的浮动元素会参与计算。

      这是利用了 BFC 的特性第 5 条实现。（可以翻回上面看看😄）

    • 布局应用（左边固定，右边自适应）

      <style>
    body {
        margin: 0;
        padding: 0;
    }
 
    .aside {
        width: 100px;
        height: 150px;
        float: left;
        background: #f66;
    }
 
    .main {
        height: 200px;
        background: #fcc;
    }
</style>
<body>
    <div class="aside">aside</div>
    <div class="main">main</div>
</body>

    ​ 根据 BFC 布局规则第3条：

    每个元素的 margin box 的左边， 与包含块 border box 的左边相接触(对于从左往右的格式化，否则相反)。即使存在浮动也是如此。

    ​

    ​ 因此，虽然存在浮动的元素 aside，但 main 的左边依然会与包含块的左边相接触。

    ​ 根据 BFC 布局规则第四条：

    BFC 的区域不会与float box（浮动元素）重叠。

    我们可以通过通过触发 main 生成 BFC， 来实现自适应两栏布局。

    .main{
	overflow: hidden;
}

    当触发 main 生成 BFC 后，这个新的 BFC 不会与浮动的 aside 重叠。因此会根据包含块的宽度，和 aside 的宽度，自动变窄。效果如下：

    可以打开 F12 看看效果哦，左右拉动屏幕大小，左侧是固定的，右侧是自适应变化的。😄

    • # 自适应两栏布局
    <!DOCTYPE html>
<html lang="en">
<head>
	<meta charset="UTF-8">
	<meta name="viewport" content="initial-scale=1.0,maximum-scale=1.0,minimum-scale=1.0,user-scalable=no">
	<title>BFC</title>
</head>
<style> 
	.left{
		background:pink;
		float: left;
		width:180px;
	}
	.center{
		background:lightyellow;
		overflow:hidden;

	}
	.right{
		background: lightblue;
		width:180px;
		float:right;
	}
</style> 

<body> 
	<div class="container">
		<div class="left">
			<p>白日依山尽</p>
			<p>黄河入海流</p>
		</div>
		<div class="right">
			<p>白日依山尽</p>
			<p>黄河入海流</p>
		</div>
		<div class="center">
			<p>超辞白帝彩云间</p>
			<p>千里江陵一日还</p>
		</div>
	</div>
</body>
</html>

    ​ 我们为了实现这个布局，让左侧块容器 左浮动 ，右侧块容器 右浮动 ，最最最关键点在中间的

    <div class="center"> ，我们设置了 overflow:hidden 让其生成一个 BFC 不被 浮动元素 覆盖，与此同时，给左右两个块设置死了宽度 width:180px ，这样在不同的分辨率视口下，就能实现左右两栏固定，中间自适应的效果。

    当然要记得在一开始就设置好 viewport 的 <meta> 元标签，不然无法实现。

    <meta name="viewport" content="initial-scale=1.0,maximum-scale=1.0,minimum-scale=1.0,user-scalable=no">

    # 总结

    ​ 以上例子都体现了一个重要特点：BFC 内部元素与其外部元素的布局，互不干扰 。因此，当 BFC 外部存在浮动元素的时候，为了不被浮动元素遮盖，会自动调节自身的大小去适应。当 BFC 内部有浮动时，为了不影响外部元素的布局，BFC 计算高度时会包括浮动的高度。 避免 margin 重叠也是这样的一个道理。

    个人觉得常用到的是 overflow: hidden 这个属性，其它的具体情况具体分析啦😜
