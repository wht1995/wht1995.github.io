    # CSS3 的变换与动画

    本文没有太多技术含量，单纯整理😄

    # 前言

    ​ 事情是这样的，某一天需要写一些 CSS 的动画，之前看过一点，觉得好像不是很难，好像可以信手捏来（谁给的自信???）。但是一上手的时候，居然不知道怎么让它动起来了，以及 transition 、transform、translate 傻傻分不清楚，过渡（transition）和动画都可以动，我该选哪个？？？之前因为自己不仔细，一直弄混和 transition、translate 的关系。

    ​ 好吧，那就来简单整理一下吧，不然真的是一片浆糊...

    ​ 那么先来个 2D 平面的吧...

    ​ 所有的 2D、3D 变换开启，需要添加到 transform 属性上，多个变换可以同时进行，用空格隔开。

    ​

    # 2D转换

    • translate

      理解：就是可以使元素 平移，沿着 X 轴和 Y 轴，有单位（px 等）。

      方法：translate(x, y) | translateX(n) | translateY(n)

    • rotate

      理解：就是可以使元素按指定的角度 旋转，单位（deg），正值按顺时针旋转，负值按逆时针旋转。

      方法：rotate(angle)

    • scale

      理解：就是可以使元素按指定的数值在 X 轴和 Y 轴上 成倍缩放，无单位。

      方法：scale(x, y) | scaleX(n) | scaleY(n)

    • skew

      理解：就是可以使元素按指定的数值在 X 轴和 Y 轴上 翻转（倾斜），单位（deg），正值按顺时针反转，负 值按逆时针翻转。

      方法：skew(x, y) | skewX(n) | skewY(n)

    • matrix

      理解：就是可以把上述四种变化 同时 加给某个元素，

      方法：matrix(n, n, n, n, n, n)

      PS：这个是一个六个值的矩阵，比较复杂，数学不好，没打算深究，用到时再说🙃目前还比较简单，示例可 以去 W3School 查看。

    # 3D转换

    ​ 前置：对于2D 来说，3D变换主要体现在 Z 轴上，Z 轴垂直于显示器屏幕，指向你自己！为了体现出 3D 的效果，我们要开启 3D 模式，创建三维空间。

    ​ 方法：perspective: none | <length> （必选项）

    ​ 理解：可以简单的理解为 视距，用来设置用户和元素 3D 空间 Z 平面之间的距离。而其效应由他的值来决定，值越小，用户与 3D 空间 Z 平面距离越近，视觉效果更大；反之，值越大，用户与3D 空间 Z 平面距离越远，视觉效果就很小。 当为元素定义 perspective 属性时，其子元素会获得透视效果，而不是元素本身。

    ​ 方法：transform-style: flat | preserve-3d （可选项）

    ​ 理解： 属性需要设置在父元素中，并且高于任何嵌套的变形元素。 该属性设置 某个元素的子元素 是以平面还是 3D 的形式展现。

    好的，理解完开启三维空间的魔法后，让我们现在进入三维空间。

    • translate

      方法：translateZ(z) | translate3d(x,y,z)

      理解：还是平移，只不过是在 Z 轴上的平移。需配合 perspective 使用， perspective 的值小于 translateZ()的值时，不产生 三维效果（透视）。**当 Z 轴值越大时，元素也离观看者更近，从视觉上元素就变得更大；反之其值越小时，元素也离观看者更远，从视觉上元素就变得更小。 **

    • rotate

      方法：rotateZ(angle) | rotate3d(x,y,z,angle)

      理解：x, y, z 的参数为 Number 类型，可以是0到1之间的数值，表示 各旋转轴坐标方向 的矢量。angle 表示旋转角度。正的角度值表示顺时针旋转，负值表示逆时针旋转。（旋转涉及到复杂的数学公式，我也解释不清，这里只作简单的了解）

    • scale

      方法：scaleZ(angle) | scale3d(x,y,z)

      理解：对于参数取值，当取值在 [-1，1] 范围时，元素沿该轴增长；在范围内时，它会收缩。如果为负，则结果在该维度上反向缩放。值为 1 时无效。 如果是 2D 平面效果，Z 轴上的变化体现为元素的放大或缩小，与视距有关（2D 平面效果需要配合 perspective 、translateZ使用）。

    # transition(过渡)

    理解：就是控制元素从当前样式变换成另一种指定样式的方法，且会逐渐变回原来的样式。和动画（animation + @keyframes）相比它是单次的变换，而动画是多次多阶段的变换。 如需向多个样式添加过渡效果，请添加多个属性，由逗号隔开。要实现这一点，必须规定两项内容：

    • 规定您希望把效果添加到哪个 CSS 属性上
    • 规定效果的时长

    方法：

    属性 可选参数 描述
    transition 以下四个皆可 简写属性，用于在一个属性中设置四个过渡属性。
    transition-property width、 height 之类的 规定应用过渡的 CSS 属性的名称。
    transition-duration 秒/毫秒，0 无效 定义过渡效果花费的时间。默认是 0。
    transition-timing-function linear、ease、ease-in、ease-out、ease-in-out 规定过渡效果的时间曲线。默认是 "ease"。
    transition-delay time( s / ms ) 规定过渡效果何时开始。默认是 0。
    <div></div>

<style> 
div
    {
        width:100px;
        height:100px;
        background:yellow;
        transition:width 2s, height 2s,transform 2s,background 2s;
    }

div:hover
    {
        width:150px;
        height:150px;
        background:pink;
        transform:rotate(30deg) translate(500px);
    }
</style>

    # animation(动画)

    理解：动画就是一连串的过渡，我自己理解为是多个 transition 的效果叠加

    方法：

    属性 可选参数 描述
    @keyframes 动画名!、动画帧!、合法CSS样式!（!为必选） 规定动画。
    animation 其余 6 种动画属性的简写属性 所有动画属性的简写属性，除了 animation-play-state 属性。
    animation-iteration-count n|infinite 规定动画被播放的次数。默认是 1。
    animation-direction normal|alternate 规定动画是否在下一周期逆向地播放。默认是 "normal"。
    animation-play-state paused|running 规定动画是否正在运行或暂停。默认是 "running"。
    animation-fill-mode none | forwards | backwards | both 规定对象动画时间之外的状态。
    <div></div>

<style> 
div
{
    width:100px;
    height:100px;
    background:#18A294;
    position:relative;
    animation:myfirst 5s;
}

@keyframes myfirst
{	
    30%{ transform:rotate(30deg);left:0px; }
    70%{ transform:rotate(0deg);left:500px;background:#FD7279  }
    100%{ transform:rotate(-360deg);left:0px;}
}
</style>

    # 总结

    CSS 的动画也是由一个一个单次的变化组合而成的，弄清楚单次的变化，知道怎么组合是实现动画的关键。

    • 开启 2D/3D 变换需要 transform 属性，多个变换用空格隔开。

    • transition 必须指定要变换的 CSS 属性还有过渡的时长。 多个样式添加过渡效果，请添加多个属性，由逗号隔开。（也可以用 all ，例： transition: all 5s）

    • 实现动画 animation 需要给 @keyframes 命名以便使用，定义好关键帧的样式变化，不能缺少动画时间。

    • 若想看到明显的 3D 效果，需要使用 perspective 配合 transform-style

    • rotate()、rotateZ()，会采用 就近转到目标角度 原则：

      a. 小于180°，正常顺时针，但大于180°就要注意：例设值 200deg，会逆时针转160度完事；

      b. 设180°会逆时针转，-180°才顺时针；

      c. 设360°的倍数值压根转都不转；

      d.rotateX() 、 rotateY() 这两个，还是会老老实实的转，一度不少，正值顺、负值逆 ；
