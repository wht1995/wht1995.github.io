# Class 的基本语法

本文整理参照阮一峰的 ECMAScript 6 入门

# 简介

​ ES6 的类，实际上是语法糖，本质上还是一个函数。类本身指向构造函数。类所有的方法都定义在类的 prototype 属性上。类的内部所有定义的方法都是 不可枚举 的。类和模块的内部，默认就是严格模式,不需要使用 use strict 指定运行模式。

# Constructor 方法

​ constructor 是默认方法，通过 new 生成实例时自动调用。一个类必须有 constructor 方法，如果没有，一个空的 constructor 会被默认添加。（最好都给写上，别整幺蛾子）

class Point {
}

// 等同于
class Point {
  constructor() {}
}

​ constructor 方法默认返回实例对象（this），也可以指定返回另一个对象

class Foo {
  constructor() {
    return Object.create(null);
  }
}

new Foo() instanceof Foo
// false

​ 上面代码中，constructor 函数返回一个全新的对象，结果导致实例对象不是 Foo 类的实例。

​ 类的所有实例 共享一个原型对象

var p1 = new Point(2,3);
var p2 = new Point(3,2);

p1.__proto__ === p2.__proto__
//true

# 取值函数（getter）和存值函数（setter）

​ getter、setter 可在类内部定义，对某个属性设置存/取值函数，可拦截该属性的存取行为。所谓拦截，就是对数据做一些处理后再进行存取

class MyClass {
  constructor() {
    // ...
  }
  get prop() {
    return 'getter';
  }
  set prop(value) {
    console.log('setter: '+value);
  }
}

let inst = new MyClass();

inst.prop = 123;
// setter: 123

inst.prop
// 'getter'

​ 上面代码中，prop 属性有对应的存值函数和取值函数，因此赋值和读取行为都被自定义了。

# 类中关于 this 的指向

​ 类中如果含有 this ，它默认指向类的实例，但是如果这个类中的方法 单独 提取出来使用，this 会指向该方法运行时所在环境（由于 class 内部都是严格模式，所以 this 指向为 undefined）

​ 解决方法：

  • bind

    class A {
  constructor() {
    this.methodName = this.methodName.bind(this);
  }

  // ...
}

  • 箭头函数（箭头函数的 this 在定义时就确定了）

    class Obj {
  constructor() {
    this.getThis = () => this;
  }
}

const myObj = new Obj();
myObj.getThis() === myObj // true

# 静态方法

​ 类相当于实例的原型，所有在类中定义的方法都会被实例继承。如果在一个方法前加上 static 关键字，就表示 该方法不会被实例继承，只能通过类本身来调用

class Foo {
  static classMethod() {
    return 'hello';
  }
}

Foo.classMethod() // 'hello'

var foo = new Foo();
foo.classMethod()
// TypeError: foo.classMethod is not a function

​ 如果静态方法中包含 this ，则this指向类，而非实例。

class Foo {
  static bar() {
    this.baz();
  }
  static baz() {
    console.log('hello');
  }
  baz() {
    console.log('world');
  }
}

Foo.bar() // hello

​ 父类的静态方法可被子类 继承 并让子类本身调用

class Foo {
  static classMethod() {
    return 'hello';
  }
}

class Bar extends Foo {
}

Bar.classMethod() // 'hello'

# 实例属性的写法

​ 可以定义在constructor() 的 this 上

class Person{
  constructor(){
    this.age = 10;
  }

  say(){
    console.log(this.age)
  }
}
let p = new Person()
p.say()  // 10

​ 也可以定义在类的最顶层（此时去掉 constructor()）

class Person{
  age = 10;

  say(){
    console.log(this.age)
  }
}
let p = new Person()
p.say()

# 静态属性

​ 是指 class 本身的属性，即 class.propName ，并非定义在实例对象（this）上的属性

class Foo {
}

Foo.prop = 1;
Foo.prop // 1
