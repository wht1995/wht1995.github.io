# Class 的继承

本文整理参照阮一峰的 ECMAScript 6 入门

# 简介

​ class 可以通过 extends 关键字实现继承

class Point {
}

class ColorPoint extends Point {
}

​ 子类在 constructor() 方法中 必须 要调用 super 方法，否则新建实例时会报错。因为子类自己的 this 对象必须先通过父类的构造函数完成塑造，得到与父类同样的实例属性和方法后，再对其加工，加上子类自己的实例属性和方法，如不调用 super ，子类就得不到 this

class Point { /* ... */ }

class ColorPoint extends Point {
  constructor() {
  }
}

let cp = new ColorPoint(); // ReferenceError

​ 上面代码 ColorPoint 继承自 Point ，但子类的构造函数没调用 super 方法，所以报错了。ES5 的继承是先创建子类的实例对象 this ，然后再将父类的方法添加到 this 上( Parent.apply(this) ),而ES6 的继承是先调用 super 获得 this ，再进行修改 this 。

​ 每个子类都会 默认添加一个 constructor() （当你未定义 constructor 时）

class ColorPoint extends Point {
}

// 等同于
class ColorPoint extends Point {
  constructor(...args) {
    super(...args);
  }
}

总结 ：子类的继承时通过 extends 实现的，子类的构造函数中，必须 调用 super 获得 this 后再对其修改。

# Object.getPrototypeOf()

​ Object.getPrototypeOf方法可以用来从子类上获取父类。

Object.getPrototypeOf(ColorPoint) === Point
// true

​ 因此，可以使用这个方法判断，一个类是否继承了另一个类。

# Super 关键字

  1. 可当作函数使用
  2. 可当作对象使用

当作函数调用时：

  • Super 作函数调用时，代表父类构造函数
  • 只能写在子类构造函数中，且 必须 执行一次，否则报错
class A {}

class B extends A {
  m() {
    super(); // 报错
  }
}
  • Super 虽然代表父类，但返回的是子类实例，即 Super 内部的 this 指向子类实例，相当于 A.prototype.constructor.call(this)
class A {
  constructor() {
    console.log(new.target.name);
  }
}
class B extends A {
  constructor() {
    super();
  }
}
new A() // A
new B() // B

用作对象时：

  • 在普通方法中指向 父类原型(Parent.Prototype) ，在静态方法中指向父类。所以定义在父类实例上的属性和方法无法通过 Super 调用。

  • ES6 规定，在子类普通方法中调用 Super 方法访问父类方法时，方法内部的 this 指向当前子类实例。

    class A {
  constructor() {
    this.x = 1;
  }
  print() {
    console.log(this.x);
  }
}

class B extends A {
  constructor() {
    super();
    this.x = 2;
  }
  m() {
    super.print();
  }
}

let b = new B();
b.m() // 2

  • 由于 this 指向子类实例，所以通过 Super 对某个属性赋值，属性就会变成子类的实例属性。

    class A {
  constructor() {
    this.x = 1;
  }
}

class B extends A {
  constructor() {
    super();
    this.x = 2;
    super.x = 3;
    console.log(super.x); // undefined
    console.log(this.x); // 3
  }
}

let b = new B();

  • 静态方法 中，Super 指向父类，并非指向父类的原型对象。在 普通方法 中才指向父类原型

  • 子类静态方法中，通过 Super 调用父类方法时，方法内部的this 指向当前子类，而并非子类实例。

    class A {
  constructor() {
    this.x = 1;
  }
  static print() {
    console.log(this.x);
  }
}

class B extends A {
  constructor() {
    super();
    this.x = 2;
  }
  static m() {
    super.print();
  }
}

B.x = 3;
B.m() // 3

总结：主要区分是否为静态方法，静态方法只和类本身有关，与类其它东西无关。
