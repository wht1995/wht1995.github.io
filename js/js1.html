# JavaScript 基础

# 模拟 new 操作符

参考链接

根据《高级程序设计》中的解释，使用 new 操作符的时候，内部发生了什么？

  • 创建一个新对象
  • 将构造函数的作用域赋给新对象（因此 this 就指向了该对象）
  • 执行构造函数中的代码，为这个新对象添加属性
  • 返回新对象

new 出来的新实例，可以：

  • 访问构造函数中的属性
  • 访问构造函数 prototype 中的属性

因为 new 是一个关键字，所以我们要另写一个函数去模拟实现。例如我们把这个函数命名为 objectFactory()

用的时候是这样的：

function Otaku(){
    ...
}
//使用 new
var person = new Otaku()

//使用 objectFactory
var person = objectFactory(Otaku, ...)

实现代码：

function objectFactory(){
    var obj = new Object();
    
    Constructor = [].shift.apply(arguments);
    
    obj.__proto__ = Constructor.prototype;
    
    var ret = Constructor.apply(obj, arguments);
    
    return typeof ret == 'object' ? ret : obj;
}
  • 用 new Object() 新建一个空对象
  • 由于用法的原因，取出第一个参数（就是要传入的构造函数）。此外因为 shift 会修改原数组，所以 arguments 会被去除第一个参数
  • 将新对象 obj 的原型指向构造函数的原型，这样 obj 就能访问到构造函数原型上的属性
  • 执行构造函数的代码，把属性赋给 obj ，改变了构造函数 this 的指向，指向了新建的 obj ,这样 obj 就能访问构造函数中的属性
  • 关于返回值做的判断，解释见参考链接

# lazyLoad

原理

  • 首先设置 <img> 标签的 src 的属性为一个默认的图片地址，例如 “default.jpg” 或者为空字符串。
  • 在 <img> 中自定义一个属性 data-src ，指向真实的需要渲染的图片地址
  • 监听滚动事件 scroll，在 scroll 事件的回调函数中判断图片是否进入了我们的可视区域
  • 一旦图片进入可视区域，就把 data-src 的值赋给 src

注： 图片要指定宽高

  let img = document.querySelectorAll("img")
  let n = 0; //记住图片加载到的位置
  
  function lazyLoad(){
    let seeHeight = document.documentElement.clientHeight; //可视区域的高度

    for(let i = n; i< img.length; i++){
      if(img[i].getBoundingClientRect().top < seeHeight){ //判断是否进入可视区域
        img[i].src = img[i].getAttribute("data-src")
        n++;
      }
    }
  }
  lazyLoad()
  window.addEventListener("scroll",lazyLoad)
