# 防抖与节流

简单来说，防抖函数与节流函数是常见的用来优化前端性能的工具函数。

比如说我们在调整窗口大小（resize）的时候、触发滚动条（scroll）的时候、当鼠标在移动（onMouseMove）的时候等，我们可以用防抖和节流对这些事件的触发频率做一些限制。

# 防抖函数

参考 link

防抖函数，称之为 debounce

用法：debounce(callback, delay)

  • callback ：期望用防抖函数来限制触发频率的回调函数
  • delay：限制频率的间隔时间，单位毫秒（ms）

顾名思义，就是防止抖动的函数。这个“抖”，我理解为你在不停的触发 resize 或 scroll 等事件，这时候防抖函数的回调函数就不会触发。当你停止触发 resize 等事件时，过了 delay 后我才会执行回调函数，而当你在 delay 期间再次触发 resize 时，我会重新计算 delay 。

举个例子：

<style>
    .box{
        height:300px;
        background-color: blueviolet;
    }
</style>

<body>
    <div class="box"></div>
</body>

var box = document.querySelector('.box');
    var n = 0;
    function console(){
        box.innerHTML = n ++
    }
box.addEventListener('mousemove', debounce(console,1000)) //下面实现debounce

页面上有一个色块，当我用鼠标在色块内移动的时候，我会看到里面的数字在 + 1。

如果没有 debounce ，数字就会疯狂的在增加，这不是我们想要的。

我们是希望，不管我怎么移动鼠标，都让他以 一秒加一次 的频率来展示。

简单实现一下：

function debounce(fn, delay){
        var timer = null;	//闭包
    
        return function(){
            clearTimeout(timer)		//每次先清理上次的定时器
            timer = setTimeout(function(){	//然后再新建一个重新计时
                fn.apply(this, arguments)	//最后在规定的间隔后执行 fn
            }, delay)
        }
    }

实现的关键在于：要 先清除 上一次的定时器，再创建 新的定时器作延迟执行。另外还需要用到一个 闭包 来记住定时器。

# 节流函数

参考 link

用法：throttle(callback, delay)

  • callback ：期望用节流函数来限制触发频率的回调函数
  • delay：限制频率的间隔时间，单位毫秒（ms）

节流函数，顾名思义就是“有节制地流动”（哈哈哈？）。就好比给水龙头装了一个装置，一次只流一滴水，不论你水龙头开多大，每次通过该装置流出来的，只有一滴！

所以话说回来，当你不断的去 resize 的时候，我也会执行回调函数，但是我不会一直执行，我会在一个设定好的时间间隔里（delay）执行，有节制的执行。

还是上面那个例子，我现在会不断去移动鼠标，在此过程中，每隔一秒，数字会累加一次。

时间戳版本：

function throttle(fn, delay){
    var previous = 0;	//闭包，记住一开始对比的时间为 0
    return function(){
        var now = new Date().getTime();	 //获取当前时间
        if(now - previous > delay){		 //如果前后时间差大于设定的 delay，就执行 fn
            fn.apply(this, arguments);
            previous = now		//把当前的时间赋值给previous，用作下一次进行对比
        }
    }
}

实现的 关键 在于：对比完时间差后，要记得把当前时间赋值给被对比的那个时间。然后，依然是用闭包来保存这个被对比的时间。

定时器版本：

function throttle(fn, delay){
    var timer = null;	//闭包，用来记住创建的定时器
    return function(){
        if(!timer){		//首先判断是否存在已经创建的定时器，若没有，则往下执行
            timer = setTimeout(function(){
                fn.apply(this, arguments)	//执行回调函数
                timer = null	//很关键，要执行完回调后，要把定时器清除
            }, delay)
        }
    }
}

实现的 关键 在于：触发事件时，首先要判断是否已经存在创建过的定时器，因为存在的话，说明在执行上一个定时器的任务。如果没有，说明上一个定时器任务完成了，或者是刚刚第一次执行节流函数，那么接下来就可以新建一个定时器了。当定时器任务执行完了，务必记得清除定时器，否则就无法创建下一个定时器。

以上只是很简单的实现了防抖&节流，还有很多可优化的地方，详细就要看看参考链接里的辣。
