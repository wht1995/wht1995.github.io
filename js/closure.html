# 闭包

闭包 是 javascript 中经常被谈论到的话题，今儿我也来梳理一下自己对闭包的理解。

先说，作用域和执行环境其实是同一个概念 。

# 📚概念

闭包是指有权访问另一个函数作用域的变量的函数。—— 红宝书（第三版）

怎么理解呢？

是时候展现一下语文功底了。

来缩句：闭包是函数

是一个什么样的函数呢 ？

是一个 可以访问另一个函数作用域 的函数。

ok，到这里就很明白了，闭包本质是一个函数，令它如此特别的原因就是，它可以访问另一个函数作用域中的变量。而普通一般的函数是不能访问自己作用域以外的变量的。

# 🌴感受闭包

function outFun() {
    var name = "Apple";
    function innerFun() {
        alert(name);
    }
    return innerFun;
}

var myFunc = outFun();
myFunc();  // Apple

这段代码中，声明了两个函数，一个是 outFun() 一个是 innerFun() ，innerFun() 写在 outFun() 内部，所以我们说 outFun() 是外部函数，在 outFun() 中我们最后把 innerFun 给 return 了出来，最后赋值给了 myFunc ，执行 myFunc ，弹框显示 Apple。

在这里，Apple 是外部函数的变量，相对内部函数 innerFun() 来说是在自己作用域之外的，按理说是访问不到的。但是这里最后能输出 Apple ，就是因为 innerFun() 形成了闭包。

对，innerFun() 就是闭包。

闭包是由函数以及创建该函数的词法环境（作用域）组合而成。这个作用域包含了这个闭包创建时所能访问的所有局部变量 。

在例子中，myFunc 是执行 outFun 时创建的 innerFun 函数实例的引用，而 innerFun 实例仍可访问其词法作用域中的变量，即可以访问到 name 。由此，当 myFunc 被调用时，name 仍可被访问，其值 Apple 就被传递到alert中。

# ❓怎么理解

当某个函数被调用时，会创建一个 执行环境（execution context）以及相应的 作用域链 。然后使用 arguments 和其他命名参数的值来初始化函数的 活动对象（activation object）。

每个执行环境（执行上下文）都有三个重要属性：

  • 变量对象(Variable object，VO)
  • 作用域链(Scope chain)
  • this

注： 变量对象在函数运行时被激活，变为 活动变量 AO（active object）

所以一个全局的执行环境是这样的，详细看 冴羽博客

globalContext = {
    VO: [global],
    Scope: [globalContext.VO],
    this: globalContext.VO
}

上面的 Scope 就是作用域链。在作用域链中，外部函数的活动对象始终处于第二位，外部函数的外部函数的活动对象处于第三位，直至作为作用域链终点的全局执行环境。

到了 innerFun 的 Scope 则是

innerFun_context = {
    Scope = [AO, outFun_copeContext.AO, globalContext.VO]
}

即每个函数的执行环境中的 Scope 都维护了一条作用域链（以全局执行环境为终点），因此它可以访问处在作用域链中的变量。也就解释了，为什么当外部函数执行完后，闭包函数仍然能够访问到外部函数的变量。

# 🌰举个例子

代码参考自 冴羽博客

var data = [];

for (var i = 0; i < 3; i++) {
  data[i] = function () {
    console.log(i);
  };
}

data[0]();
data[1]();
data[2]();

结果，都是输出 3 。

这三个函数在循环中被创建，但是他们共享一个词法作用域，就是全局作用域，这个作用域中存在一个变量 i ，当 data[i] 要访问 i 时，会沿着作用域链向上找，找到属于全局作用域的 i 。由于循环在 data[0]() 执行之前就已经执行完毕了，所以变量 i 已经变为 3 了，所以都输出 3 。

为了解决这个问题，我们可以使用再多一个闭包（其实这样并不那么好）

var data = [];

for (var i = 0; i < 3; i++) {
  data[i] = (function (i) {
        return function(){
            console.log(i);
        }
  })(i);
}

data[0](); // 0
data[1](); // 1
data[2](); // 2

这里我们看到，在原来的代码外又创建了一个闭包，此时作用域链是：

return function() 的作用域 ==> 匿名函数的作用域 ==> 全局作用域。

这段代码中，所有创建的 data[i] 不再共享同一个环境，匿名函数为每一个 data[i] 又创建了一个新的词法环境。在这些环境中，i 指向对应循环的 i 。

即执行 data[0] 的时候，data[0] 的函数作用域是这样的：

data[0]Context = {
    Scope: [AO, 匿名函数Context.AO globalContext.VO]
}

我们并没有把闭包直接赋值给数组，而是定义了一个立即执行的匿名函数，把匿名函数的结果赋值给数组。匿名函数接受 i 作为参数（形参写啥都行其实），也就是最终要打印出来的值。在调用每个匿名函数时，我们把变量 i 传入。由于函数参数是按值传递的，所以就会将变量 i 的当前值复制给匿名函数的参数 i 。而在这个 匿名函数内部，又创建并返回了一个访问 i 的闭包。这样一来，data[] 数组中的每个函数都有自己 i 变量的一个副本，因此就可以返回各自不同的数值了。

# 🍉模拟私有方法

var Counter = (function() {
  var privateCounter = 0;
  function changeBy(val) {
    privateCounter += val;
  }
  return {
    increment: function() {
      changeBy(1);
    },
    decrement: function() {
      changeBy(-1);
    },
    value: function() {
      return privateCounter;
    }
  }   
})();

console.log(Counter.value()); /* logs 0 */
Counter.increment();
Counter.increment();
console.log(Counter.value()); /* logs 2 */
Counter.decrement();
console.log(Counter.value()); /* logs 1 */

清晰易懂，这段代码用 Counter 接收了一个匿名函数的返回值，匿名函数返回了一个对象。里面有三个方法，

这三个方法可以通过 Counter 来访问，实现了私有化。而别的一般方式访问这三个方法是不可行的。
